Introduction to Roguelikes

What is a roguelike? First of all, it is my favorite genre of game! Second of all, this is a question that has puzzled enjoyers of the genre for years, as the term "Roguelike" gets applied to everything from the original Rogue to Fortnite.

What is a Roguelike?

Developers and players at the International Roguelike Development Conference of 2008 argued for many hours about what defines a Roguelike, eventually deciding on 13 key factors, known as the Berlin Interpretation:

  1. The world of the game is randomly generated
  2. Death is permanent; you must create a new character after you die if you want to play again
  3. The game is turn based and grid based (like a board game)
  4. There is no meta progression; each new start is a blank slate with no impact from previous runs
  5. Actions are non-modal. You should be able to do any action regardless of game state
    (e.g, choosing to fight in a shop)
  6. The game is complex enough to allow multiple solutions to the same problem (more on this later!)
  7. You have to manage your limited resources (e.g. food, healing potions)
    and find uses for the resources you receive
  8. Killing hoards of enemies must be a part of gameplay
  9. The world must be explored carefully by the player (unless you want to become familiar with rule 2)
  10. The player controls a single character and the game is singleplayer
  11. Enemies abide by the same rules as players (They can cast fireballs and detonate bombs too,
    watch out!)
  12. The game is a tactical challenge, you have to learn and plan before you can make significant progress
  13. Graphics are minimal

Naturally, much has changed since 2008, and these rules are now seen as guidelines rather than the law. Games that violate these rules (to a point) are seen as roguelikes by many:

  • The Binding of Isaac
  • Vampire Survivors
  • Rogue Legacy
  • Brotato
  • Inscryption
